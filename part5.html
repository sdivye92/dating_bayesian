The Full Pipeline: Prior, Likelihood, MCMC, and Decision

Complete Python implementation of the hierarchical Bayesian Skew-Normal dating decision model, with full annotation of every numerical choice, Program 1 (Bayesian EU Calculator), Program 2 (Decision Surface), and Program 3 (Full MCMC Pipeline).

Parts I through IV developed the mathematical structure of the model. This part implements it. Three programs are provided. Program 1 computes Bayesian expected utility from posterior samples using Monte Carlo integration. Program 2 generates the decision surface plot showing where confession is rational as a function of \(\xi\) and \(\Delta\). Program 3 implements the full end-to-end pipeline: prior specification, likelihood model, Metropolis-Hastings MCMC posterior sampling, and final decision output. Every numerical choice is annotated with its justification.

§1 Full Algorithmic Pipeline Summary

8-Step Implementation Pipeline
  1. Define payoff matrix: Set \(u_{CA}, u_{CB}, u_{CR}, u_{DA}, u_{DB}, u_{DR}\).
  2. Specify prior \(P(\xi, \Delta)\): Bivariate Gaussian with mean \(\mu_0\) and covariance \(\Sigma_0\), truncated to \(\Delta > 0\).
  3. Specify likelihood model \(P(E \mid \xi, \Delta)\): Per-signal conditional distributions (logistic, log-Normal, Gaussian).
  4. Construct unnormalized posterior: \(\tilde{P}(\xi, \Delta \mid E) = P(E \mid \xi, \Delta)\,P(\xi, \Delta)\).
  5. Sample posterior using MCMC: Run Metropolis-Hastings with symmetric Gaussian proposal.
  6. Evaluate utility per posterior sample: Apply \(U_C(\xi^{(i)}, \Delta^{(i)})\) and \(U_D(\xi^{(i)}, \Delta^{(i)})\) for each sample.
  7. Monte Carlo average utilities: \(\mathrm{EU}_C \approx \frac{1}{N}\sum_i U_C(\xi^{(i)}, \Delta^{(i)})\), similarly for \(\mathrm{EU}_D\).
  8. Compare and decide: Confess iff \(\mathrm{EU}_C > \mathrm{EU}_D\).

§2 Program 1: Bayesian Expected Utility Calculator

This program takes posterior samples over \((\xi, \Delta)\) (either from MCMC or from a specified posterior distribution) and computes the Monte Carlo-approximated expected utilities for both actions.

// program1_bayesian_eu_calculator.py
# ============================================================
# Program 1: Bayesian Expected Utility Calculator
# ============================================================
#
# This implements the core EU computation from posterior samples.
# The key insight: we do NOT plug in posterior means.
# We average the utility function over all posterior samples,
# which correctly accounts for nonlinearity in g(xi, delta).
#
# NUMBER JUSTIFICATIONS (Payoff Matrix):
#
#   u_CA = +10: Maximum utility. Confession reciprocated.
#               Resolves uncertainty, creates relationship,
#               rewards risk-taking. Asymmetrically high.
#
#   u_CB =  -3: Benching. Emotional limbo: hope without
#               commitment. Investment without reciprocity.
#               Negative but less so than rejection because
#               at least interest exists.
#
#   u_CR =  -8: Full rejection. Social risk taken and lost.
#               Ego cost, awkwardness, possible relationship
#               damage. Large magnitude to reflect genuine
#               psychological cost.
#
#   u_DA =  -6: Missed opportunity. Would-have-accepted but
#               confessor stayed silent. Regret. Magnitude
#               close to u_CR because missed reciprocation
#               can be nearly as painful as rejection.
#
#   u_DB =  -2: Continued ambiguity. In the bench orbit
#               without resolution. Mild but persistent cost.
#
#   u_DR =  +1: Correct avoidance. Saved from rejection pain.
#               Small positive: dignity preserved.
#
# These values produce the 16pA - pB - 9pR criterion after
# differencing, which correctly weights acceptance heavily
# and penalizes rejection over benching.
# ============================================================

import numpy as np
from scipy.stats import skewnorm
from scipy.integrate import quad
from scipy.special import ndtr as Phi   # Standard normal CDF

# ---- Payoff matrix ----
U_CA, U_CB, U_CR = 10, -3, -8
U_DA, U_DB, U_DR = -6, -2,  1

# ---- Fixed hyperparameters ----
# omega = 1.0: Unit scale. Perception noise normalized to 1.
#              This is the standard deviation of the evaluator's
#              internal valuation draw X | xi ~ SN(xi, omega, alpha).
#              omega = 1 means a 1-unit shift in xi corresponds to
#              a 1-sigma shift in the desirability draw.
#
# alpha = -1.5: Left-skewed market (alpha < 0).
#               Most common in evaluators with high option abundance:
#               their perception of candidates compresses toward the
#               lower end, with few individuals clearly exceeding
#               their commitment threshold.
#               |alpha| = 1.5 is moderate; values > 3 would imply
#               very extreme market asymmetry.
OMEGA = 1.0
ALPHA = -1.5


def owens_T(h, a):
    """Owen's T-function via numerical integration.
    T(h, a) = (1/2pi) * integral_0^a exp(-h^2*(1+x^2)/2) / (1+x^2) dx
    """
    def integrand(x):
        return np.exp(-0.5 * h**2 * (1 + x**2)) / (1 + x**2)
    val, _ = quad(integrand, 0, a)
    return val / (2 * np.pi)


def decision_function_g(xi, delta, omega=OMEGA, alpha=ALPHA):
    """
    Closed-form g(xi, delta) = EU_C - EU_D under Skew-Normal.
    Derived in Part III:
      g = 16 - 17*Phi((delta-xi)/omega) - 8*Phi(-xi/omega)
            + 34*T((delta-xi)/omega, alpha) + 16*T(-xi/omega, alpha)
    
    Confess iff g > 0.
    """
    h1 = (delta - xi) / omega    # Standardized distance to accept threshold
    h2 = -xi / omega             # Standardized distance to bench threshold
    
    phi1 = Phi(h1)
    phi2 = Phi(h2)
    T1 = owens_T(h1, alpha)
    T2 = owens_T(h2, alpha)
    
    return 16 - 17*phi1 - 8*phi2 + 34*T1 + 16*T2


def region_probs_skewnorm(xi, delta, omega=OMEGA, alpha=ALPHA):
    """Compute P(Accept), P(Bench), P(Reject) from SN CDF."""
    dist = skewnorm(alpha, loc=xi, scale=omega)
    p_R = dist.cdf(0)
    p_A = 1 - dist.cdf(delta)
    p_B = dist.cdf(delta) - dist.cdf(0)
    return p_A, p_B, p_R


def utility_confess(xi, delta, **kwargs):
    """U_C(xi, delta): piecewise constant from payoff matrix."""
    p_A, p_B, p_R = region_probs_skewnorm(xi, delta, **kwargs)
    return U_CA * p_A + U_CB * p_B + U_CR * p_R


def utility_dont_confess(xi, delta, **kwargs):
    """U_D(xi, delta): piecewise constant from payoff matrix."""
    p_A, p_B, p_R = region_probs_skewnorm(xi, delta, **kwargs)
    return U_DA * p_A + U_DB * p_B + U_DR * p_R


def bayesian_eu_from_samples(samples):
    """
    Compute E[EU_C | E] and E[EU_D | E] via Monte Carlo.
    samples: array of shape (N, 2), columns are (xi, delta).
    
    Key: We use the utility function (which internally uses SN CDF)
    not the closed-form g, so we can separately track EU_C and EU_D.
    """
    eu_c_vals = np.array([utility_confess(xi, delta) for xi, delta in samples])
    eu_d_vals = np.array([utility_dont_confess(xi, delta) for xi, delta in samples])
    
    eu_c = np.mean(eu_c_vals)
    eu_d = np.mean(eu_d_vals)
    
    return {
        "E[EU_C | E]"    : eu_c,
        "E[EU_D | E]"    : eu_d,
        "E[g | E]"       : eu_c - eu_d,
        "Decision"       : "CONFESS" if eu_c > eu_d else "DON'T CONFESS",
        "MC_std_EU_C"    : np.std(eu_c_vals) / np.sqrt(len(samples)),
        "MC_std_EU_D"    : np.std(eu_d_vals) / np.sqrt(len(samples)),
    }

§3 Program 2: Decision Surface Plot

// program2_decision_surface.py
# ============================================================
# Program 2: Decision Surface Plot for xi and Delta
# ============================================================
#
# Plots EU_C - EU_D as a function of xi and Delta.
# Green regions: confess. Red regions: don't confess.
# Black contour: decision boundary where EU_C = EU_D.
#
# This is the POINT-ESTIMATE version of the surface
# (no posterior uncertainty). It shows what g(xi, delta)
# looks like deterministically, which gives intuition
# before adding the Bayesian averaging layer.
# ============================================================

import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import skewnorm

U_CA, U_CB, U_CR = 10, -3, -8
U_DA, U_DB, U_DR = -6, -2,  1
OMEGA = 1.0
ALPHA = -1.5

def decision_value_point(xi, delta):
    dist = skewnorm(ALPHA, loc=xi, scale=OMEGA)
    p_R = dist.cdf(0)
    p_A = 1 - dist.cdf(delta)
    p_B = dist.cdf(delta) - dist.cdf(0)
    eu_c = U_CA*p_A + U_CB*p_B + U_CR*p_R
    eu_d = U_DA*p_A + U_DB*p_B + U_DR*p_R
    return eu_c - eu_d

# Grid: xi from -2 to 5 (covers below-threshold to well-above-threshold)
#       Delta from 0.1 to 5 (must be positive; covers low to high selectiveness)
xi_vals    = np.linspace(-2, 5, 200)
delta_vals = np.linspace(0.1, 5, 200)
XI, DELTA  = np.meshgrid(xi_vals, delta_vals)
Z = np.vectorize(decision_value_point)(XI, DELTA)

fig, ax = plt.subplots(figsize=(10, 7))

cf = ax.contourf(XI, DELTA, Z, levels=40, cmap='RdYlGn')
plt.colorbar(cf, ax=ax, label=r'$\mathrm{EU}_C - \mathrm{EU}_D$')

# Decision boundary: EU_C = EU_D
ax.contour(XI, DELTA, Z, levels=[0], colors='black', linewidths=2.5)

ax.set_xlabel(r'$\xi$ (Relative Standing above Bench Threshold)', fontsize=12)
ax.set_ylabel(r'$\Delta$ (Commitment Gap / Selectiveness)', fontsize=12)
ax.set_title('Bayesian Confession Decision Surface\n'

Figure 1 Decision surface \(g(\xi, \Delta) = \mathrm{EU}_C - \mathrm{EU}_D\) over the parameter space. Green regions indicate confession is rational; red regions indicate silence is rational. How to Read the Decision Surface

  • Green region (EU_C > EU_D): Confession is rational. Corresponds to high \(\xi\) (strong positive signals) and low \(\Delta\) (low selectiveness).
  • Red region (EU_C < EU_D): Silence is rational. Corresponds to low \(\xi\) (weak signals) or high \(\Delta\) (very selective evaluator).
  • Black boundary: The indifference curve where \(\mathrm{EU}_C = \mathrm{EU}_D\). This is the decision boundary. Confession is rational to the right of and below this curve.
  • Shape insight: The boundary slopes upward from left to right: a higher \(\xi\) is needed to compensate for higher \(\Delta\). A very selective evaluator requires substantially stronger evidence of attraction before confession becomes rational.

§4 Program 3: Full MCMC Pipeline

The likelihood model in the previous programs used a single Gaussian observation model where each behavioral signal is treated as a noisy scalar estimate of \(\xi\). This is a reasonable simplification but conflates qualitatively different kinds of evidence. A binary signal (did she initiate the conversation?) has a fundamentally different structure from a continuous one (how long did she take to reply?). A commitment-avoidance statement updates beliefs about \(\Delta\) (the evaluator's selectiveness), not about \(\xi\) (the confessor's relative standing). The program below separates these signal types into four distinct likelihood models, each grounded in the Part IV formulations.

Signal-Specific Likelihood Models and Parameter Justification

Binary attraction signals — e.g., does she initiate conversations unprompted? Does she remember and bring up personal details you shared? Does she seek proximity without obvious reason? These are Bernoulli observations whose probability increases with \(\xi\) via a logistic link:

Binary Attraction Likelihood \[ P(e_i = 1 \mid \xi) = \sigma(\beta_{\text{attr}} \cdot \xi), \quad \sigma(x) = \frac{1}{1+e^{-x}} \]

Parameter choice: \(\beta_{\text{attr}} = 1.2\). At \(\xi = 0\) (exactly at bench threshold), a positive signal has probability \(\sigma(0) = 0.5\), i.e., coin-flip odds. At \(\xi = 1\), probability rises to \(\sigma(1.2) \approx 0.77\). At \(\xi = 2\), probability is \(\sigma(2.4) \approx 0.92\). The coefficient is chosen above 1 so each observation is genuinely informative, but not so high that a single observation collapses the posterior to near-certainty. In dating terms: someone who is clearly into you does not initiate 100% of the time. Life is noisy.

Response time signals — reply latency in minutes is a continuous, positive signal. High \(\xi\) (genuine interest) lowers expected response time. Log-Normal is the appropriate model because reply times are strictly positive and right-skewed:

Response Time Likelihood (Log-Normal) \[ \log e_k \sim \mathcal{N}(\gamma_0 - \gamma_1 \xi,\; \tau^2) \]

Parameter choices: \(\gamma_0 = 2.0\) gives a baseline expected log-time of 2, i.e., \(e^2 \approx 7\) minutes for an evaluator who is at the bench threshold. \(\gamma_1 = 0.4\) means each unit increase in \(\xi\) reduces expected log-time by 0.4 units, roughly halving the expected reply time at \(\xi = 2\) compared to \(\xi = 0\). The noise \(\tau = 0.5\) reflects that reply times also depend on busyness, phone availability, and context. In dating terms: someone who is into you replies faster on average, but not always faster on every single message.

Continuous warmth scores — self or observer-assessed ratings of warmth, openness, and emotional engagement on a standardised scale. Treated as direct noisy measurements of \(\xi\):

Warmth Score Likelihood (Gaussian) \[ e_i \mid \xi \sim \mathcal{N}(\xi,\; \sigma_e^2), \quad \sigma_e = 0.8 \]

The observation noise \(\sigma_e = 0.8\) is chosen to be less than the prior standard deviation \(\sigma_\xi = 1.5\), so each warmth observation meaningfully updates the posterior. It is not chosen to be very small (e.g., 0.2) because warmth ratings are subjective and context-dependent. In dating terms: how warm she seemed in Tuesday's conversation is a genuine signal about how she values you, but one conversation cannot fully resolve the question.

Binary commitment-avoidance signals — statements like "I'm not ready for a relationship," mentioning other people she is seeing, or expressing that she needs to focus on career or personal growth. These are Bernoulli observations that update beliefs about \(\Delta\) (selectiveness), not \(\xi\):

Commitment-Avoidance Likelihood \[ P(e_j = 1 \mid \Delta) = \sigma(\beta_{\text{commit}} \cdot \Delta), \quad \beta_{\text{commit}} = 0.8 \]

Parameter choice: \(\beta_{\text{commit}} = 0.8\), slightly lower than \(\beta_{\text{attr}}\), because commitment-avoidance language is noisier — people say "I'm not ready" both when they genuinely have a wide \(\Delta\) and as a reflexive social phrase. At \(\Delta = 2.0\) (the prior mean), the probability of hearing an avoidance signal is \(\sigma(1.6) \approx 0.83\). This is the key structural asymmetry in the model: avoidance signals update \(\Delta\) while attraction signals update \(\xi\). They are informationally different and treated as such.

// program3_full_mcmc_pipeline.py
# ============================================================
# Program 3: Full End-to-End Bayesian MCMC Pipeline
# with Signal-Specific Likelihood Models
# ============================================================
#
# PRIOR JUSTIFICATIONS:
#   mu_xi    = 0.0 : Neutral. No information about standing.
#                    xi=0 means exactly at the bench threshold.
#   mu_delta = 2.0 : Moderate commitment gap. Most evaluators
#                    require meaningful margin above baseline
#                    liking before committing.
#   sigma_xi = 1.5 : High prior uncertainty. We genuinely do
#                    not know where we stand before observing.
#   sigma_d  = 1.0 : Moderate uncertainty on selectiveness.
#
# LIKELIHOOD PARAMETER JUSTIFICATIONS:
#   beta_attraction = 1.2 : At xi=0 -> P(positive binary signal)=0.5
#                           At xi=1 -> P=0.77. Informative but not
#                           deterministic. Someone into you still
#                           doesn't initiate every single time.
#   beta_commitment = 0.8 : Softer sensitivity for avoidance signals.
#                           Avoidance language is noisier; people
#                           say "I'm not ready" reflexively too.
#   gamma_0 = 2.0  : Baseline log-reply-time (e^2 ~ 7 min) at xi=0.
#   gamma_1 = 0.4  : Each unit of xi halves expected reply latency.
#   tau_rt  = 0.5  : Log-response-time noise. Replies also depend
#                    on busyness, context, phone availability.
#   sigma_e = 0.8  : Warmth score noise. Less than sigma_xi so
#                    observations update beliefs meaningfully.
#
# MCMC JUSTIFICATIONS:
#   n_samples   = 5000 : SE on EU typically < 0.05 utility units.
#   burn_in     = 500  : Chain reaches posterior mode before sampling.
#   proposal_std= 0.3  : Targets 25-40% acceptance. For this
#                        posterior geometry 0.3 gives ~73% which
#                        is slightly high; reduce to 0.5 for
#                        better mixing in production.
# ============================================================

import numpy as np
from scipy.stats import skewnorm
from scipy.special import expit   # sigmoid function

U_CA, U_CB, U_CR = 10, -3, -8
U_DA, U_DB, U_DR = -6, -2,  1
OMEGA, ALPHA = 1.0, -1.5

BETA_ATTRACTION = 1.2   # binary attraction signal sensitivity
BETA_COMMITMENT = 0.8   # commitment-avoidance signal sensitivity
GAMMA_0         = 2.0   # baseline log-response-time
GAMMA_1         = 0.4   # xi effect on log-response-time
TAU_RT          = 0.5   # log-response-time noise
SIGMA_E         = 0.8   # warmth score noise


def log_likelihood(signals, xi, delta):
    """
    Signal-specific log-likelihood.
    signals: dict with keys:
      'attraction_binary' : list of 0/1  (initiated conversation, remembered details)
      'response_times'    : list of floats in minutes (reply latency)
      'warmth_scores'     : list of floats in [-3,3] (warmth rating)
      'avoidance_binary'  : list of 0/1  (commitment-avoidance statements)
    """
    log_l = 0.0

    # 1. Binary attraction signals: update xi
    #    e ~ Bernoulli(sigmoid(beta_attr * xi))
    for e in signals.get('attraction_binary', []):
        p = np.clip(expit(BETA_ATTRACTION * xi), 1e-9, 1-1e-9)
        log_l += e * np.log(p) + (1-e) * np.log(1-p)

    # 2. Response time signals: update xi
    #    log(e) ~ N(gamma_0 - gamma_1*xi, tau^2)
    #    Higher xi = shorter expected reply time
    for e in signals.get('response_times', []):
        if e > 0:
            mu_rt = GAMMA_0 - GAMMA_1 * xi
            log_l += -0.5 * ((np.log(e) - mu_rt) / TAU_RT) ** 2

    # 3. Warmth scores: update xi
    #    e ~ N(xi, sigma_e^2)
    for e in signals.get('warmth_scores', []):
        log_l += -0.5 * ((e - xi) / SIGMA_E) ** 2

    # 4. Commitment-avoidance signals: update delta
    #    e ~ Bernoulli(sigmoid(beta_commit * delta))
    for e in signals.get('avoidance_binary', []):
        p = np.clip(expit(BETA_COMMITMENT * delta), 1e-9, 1-1e-9)
        log_l += e * np.log(p) + (1-e) * np.log(1-p)

    return log_l


def log_prior(xi, delta):
    if delta <= 0: return -np.inf
    return (-0.5*((xi    - 0.0)/1.5)**2
            -0.5*((delta - 2.0)/1.0)**2)


def log_posterior(signals, xi, delta):
    lp = log_prior(xi, delta)
    return lp + log_likelihood(signals, xi, delta) if not np.isinf(lp) else -np.inf


def mcmc(signals, n=5000, burn=500, pstd=0.3, seed=42):
    rng = np.random.default_rng(seed)
    xi, delta = 0.0, 2.0
    lp = log_posterior(signals, xi, delta)
    samples, acc = [], 0
    for i in range(n + burn):
        xi2 = xi    + rng.normal(0, pstd)
        d2  = delta + rng.normal(0, pstd)
        lp2 = log_posterior(signals, xi2, d2)
        if np.log(rng.uniform()) < lp2 - lp:
            xi, delta, lp = xi2, d2, lp2
            if i >= burn: acc += 1
        if i >= burn: samples.append([xi, delta])
    return np.array(samples), acc / n


def compute_eu(samples):
    eu_c, eu_d = [], []
    for xi, d in samples:
        dist = skewnorm(ALPHA, loc=xi, scale=OMEGA)
        pR = dist.cdf(0); pA = 1 - dist.cdf(d); pB = dist.cdf(d) - dist.cdf(0)
        eu_c.append(U_CA*pA + U_CB*pB + U_CR*pR)
        eu_d.append(U_DA*pA + U_DB*pB + U_DR*pR)
    ec, ed = np.mean(eu_c), np.mean(eu_d)
    return ec, ed, "CONFESS" if ec > ed else "DON'T CONFESS"


def pipeline(label, signals):
    print(f"\n{'='*60}\n  {label}\n{'='*60}")
    samples, acc = mcmc(signals)
    xi_m, d_m = np.mean(samples[:,0]), np.mean(samples[:,1])
    dist = skewnorm(ALPHA, loc=xi_m, scale=OMEGA)
    pA = 1-dist.cdf(d_m); pR = dist.cdf(0); pB = dist.cdf(d_m)-dist.cdf(0)
    ec, ed, dec = compute_eu(samples)
    print(f"  Posterior xi    : {xi_m:+.3f} ± {np.std(samples[:,0]):.3f}")
    print(f"  Posterior Delta : {d_m:+.3f} ± {np.std(samples[:,1]):.3f}")
    print(f"  P(Accept|E)     : {pA:.3f}")
    print(f"  P(Bench |E)     : {pB:.3f}")
    print(f"  P(Reject|E)     : {pR:.3f}")
    print(f"  E[EU_C|E]       : {ec:+.4f}")
    print(f"  E[EU_D|E]       : {ed:+.4f}")
    print(f"  E[g|E]          : {ec-ed:+.4f}")
    print(f"  Decision        : {dec}")


if __name__ == "__main__":

    # Scenario A: Clear Disinterest
    # Never initiates, slow replies (45-90 min), cold warmth, stated not looking
    pipeline("Scenario A: Clear Disinterest", {

Scenario A: Clear Disinterest

The posterior places \(\xi\) at \(-2.52\), two and a half units below the bench threshold. Three consecutive failures to initiate, paired with 45-to-90 minute reply times and cold warmth scores, push the standing estimate deep into the rejection region. The avoidance signal contributes a mild upward nudge to \(\Delta\) but it barely matters here, because \(\xi\) is so low that outcome probabilities collapse entirely to rejection. P(Reject|E) = 1.000. The expected utility of confessing is \(-7.99\), which is essentially the rejection payoff \(u_{CR} = -8\) with near-certainty. The model is unambiguous: the evidence constitutes a sustained pattern of disengagement and silence is the rational choice.

Scenario B: Friendly But Ambiguous

This is the most common and most psychologically difficult scenario. The posterior \(\xi = -0.18\) sits just below zero: slightly below the bench threshold. She initiates two out of three times. She replies in 8-to-20 minutes. Her warmth scores are mildly positive. By naive reading, she seems friendly. But the model correctly identifies that friendly-but-not-invested is precisely the bench-threshold profile. P(Reject|E) = 0.877. The 12% bench probability represents the realistic possibility that she is keeping options open without intending to commit. The expected decision value is \(-7.44\), strongly negative. The model says: do not confess. The evidence looks warm but does not clear even the bench threshold in expectation. This result formalises the intuition that "she's friendly to everyone" is not the same as "she's into you."

Scenario C: Strong Positive Signals, No Avoidance

The posterior \(\xi = +1.68\) sits well above the bench threshold. She always initiates. She replies in 2-to-5 minutes. Warmth scores are strong. No avoidance signals. The prior on \(\Delta\) is 2.0 but the absence of avoidance signals pulls the posterior \(\Delta\) down slightly to 1.49. Given \(\xi = 1.68 > \Delta = 1.49\), there is a 27.1% posterior probability of acceptance. The bench probability of 63.6% reflects that even under strong signals, uncertainty remains. What shifts the decision is the expected utility calculation: E[EU_C|E] = +1.07 against E[EU_D|E] = -3.08. The regret cost of not acting when acceptance is this plausible (\(u_{DA} = -6\), weighted by P(A|E) = 0.27) outweighs the residual rejection risk. E[g|E] = +4.15: confess.

Scenario D: Strong Attraction But Highly Selective

This is the most instructive scenario because attraction signals are identical to Scenario C, yet the decision is much closer. She always initiates, replies fast, and is warm. But she has stated twice that she is not ready for a relationship. These two avoidance signals push the posterior \(\Delta\) up to 2.16 (from the prior mean of 2.0). The posterior \(\xi = +1.68\) is now below \(\Delta = 2.16\), which means P(Accept|E) = 0.054 while P(Bench|E) = 0.854. She almost certainly likes him. She almost certainly will not commit right now. The expected utility of confessing is \(-1.09\) and of not confessing is \(-2.42\). Both are negative. This is the defining feature of the high-selectiveness regime: there is no good option. But confessing is less bad. E[g|E] = +1.33, and the decision is confess, driven by the logic that not confessing under near-certain benching still carries regret cost \(u_{DB} = -2\) at high probability. The model correctly captures the "she likes me but is not ready" situation: confessing may not produce the hoped-for outcome, but staying silent almost certainly produces neither clarity nor forward movement.

§7 The Posterior Covariance Structure

The posterior over \((\xi, \Delta)\) after observing evidence is not just a point; it is a distribution. Its covariance structure carries important information:

Posterior Covariance Interpretation \[ \Sigma_{\mathrm{post}} = \begin{bmatrix} \mathrm{Var}(\xi \mid E) & \mathrm{Cov}(\xi, \Delta \mid E) \\ \mathrm{Cov}(\xi, \Delta \mid E) & \mathrm{Var}(\Delta \mid E) \end{bmatrix} \]

\(\mathrm{Var}(\xi \mid E)\): Residual uncertainty about relative standing after observing all signals. Decreases as more signals are observed. In Scenario A and C, \(\sigma_\xi \approx 0.40\), considerably narrower than the prior width of 1.5, reflecting that five or more signals have substantially resolved standing.

\(\mathrm{Var}(\Delta \mid E)\): Residual uncertainty about the evaluator's selectiveness. Remains near the prior width of 1.0 in most scenarios because selectiveness signals (commitment avoidance statements, observable option abundance) are rare. \(\Delta\) uncertainty persists longer than \(\xi\) uncertainty.

\(\mathrm{Cov}(\xi, \Delta \mid E)\): How learning about standing updates beliefs about selectiveness. Under the independent prior and current likelihood design, this covariance is largely determined by the prior. A richer model would allow attraction signals to carry weak information about selectiveness: an evaluator who responds enthusiastically is perhaps less selective than one who responds warmly but guardedly.

§8 Final Conceptual Summary

Full Decision Chain \[ E = \{e_1, \ldots, e_n\} \;\xrightarrow{\text{Bayes}}\; P(\xi, \Delta \mid E) \;\xrightarrow{\text{MC}}\; \mathrm{EU}_C,\, \mathrm{EU}_D \;\xrightarrow{\text{compare}}\; a^* \in \{C, D\} \]

The framework replaces the folk heuristic "she seems into me" with the formally precise question: given all observable evidence, is the posterior-weighted expected utility of confessing greater than the posterior-weighted expected cost of silence?

The Formal Question
\[ \text{Confess iff } \mathbb{E}[g(\xi, \Delta) \mid E] > 0 \]
where   g = 16 − 17Φ((Δ−ξ)/ω) − 8Φ(−ξ/ω) + 34T((Δ−ξ)/ω,α) + 16T(−ξ/ω,α)

George Box's observation that all models are wrong but some are useful applies here with particular force. Romantic attraction is among the most complex, context-dependent, and culturally shaped of human phenomena. No finite set of latent variables, threshold rules, and likelihood functions can fully capture it. The signal types proposed here — binary initiation, response latency, warmth scores, avoidance statements — are reasonable proxies but they are proxies. A person's actual confession decision involves information that resists numerical encoding: tone of voice, eye contact, months of shared history, intuitions that cannot be articulated.

What the model does is make one specific aspect of the decision structure explicit: the tradeoff between the risk of rejection, the cost of regret, and the probability of benching under a selective evaluator. It forces the question of what evidence is actually being collected, what that evidence tells you about the evaluator's standing and selectiveness, and whether the expected utility of acting exceeds the expected utility of waiting. That is a narrower and more tractable question than "should I confess," and it is the question the mathematics is actually answering.

The evidence sets, payoff values, signal sensitivities, and prior parameters used here reflect one set of modelling choices. Someone with different intuitions about how informative response times are, or different beliefs about the baseline selectiveness of the evaluators they encounter, or different personal utilities for benching versus rejection, should adjust the parameters accordingly. The framework accommodates all of these adjustments without structural change. It is a template, not a verdict.

On the signal types: The four likelihood models here are illustrative. A practitioner extending this framework might add ordinal signals (e.g., how many consecutive days of conversation), categorical signals (meeting contexts: one-on-one vs. group), or temporal decay weights so recent evidence counts more than old evidence.

On Gibbs vs Metropolis-Hastings: Gibbs sampling requires tractable conditional distributions \(P(\xi \mid \Delta, E)\) and \(P(\Delta \mid \xi, E)\). With logistic and log-Normal observation models, these are not available in closed form. Metropolis-Hastings with a symmetric Gaussian proposal is the simplest correct alternative. HMC/NUTS (PyMC, Stan) is preferred for production use due to better mixing in correlated posteriors.

On extending the model: Natural extensions include time-decaying evidence, dynamic per-signal updating without full re-sampling, two-sided Bayesian inference where the evaluator is also making inference about the confessor's intent, and repeated-game extensions where rejection updates future priors.
